Core System Integration

This document details how we bridge server settings with the live game world during the GameLauncher boot sequence.

Settings & Config

The Settings class provides the static constants required for the server to determine its behavior. We parse these from the main(String[] args) array.

  • WORLD_ID: Unique identifier for the instance.
  • DEBUG: Enables verbose logging and developer commands.
  • SPAWN_WORLD: Toggles the economy between standard and "Instant PVP" modes.

World Lifecycle

Initialization World.init() is called. We prepare the region manager and entity lists.
Active Loop We use World.getPlayers() every 60 seconds to perform automated background saves.
Termination During processShutdown, we iterate the world to force-disconnect and save all sessions.
Technician Note: We must ensure Settings.init() is called before any other manager. If the world starts before the settings are loaded, we risk initializing the cache with incorrect paths or memory limits.
